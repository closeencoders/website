Trying To Get Into Game Dev as a Noob

2026-03-11

I want to make games and know how they work. Not just generate code and assets with an AI or let a modern engine do all the work. I want to at least understand what’s happening under the hood enough to feel like I’m getting my hands dirty before using more advanced tools.

My hope is other people do too, and if someone reads this mabye they can learn from my mistakes.

In a previous life, I tried to make a few games in Java, but it always felt clunky, like the language was fighting me. Java would scream, "WHERE is my Jira ticket number and business requirements!?" as I forced it to loop through entities in some static factory thing. Im sure Java is great for this for others, but not me.

I wondered if making games was just not for me for a while, but the itch never really went away. It just looks so cool and rewarding.

So I am trying to make the switch to another language that will still let me mess around with Raylib and get the hang of the simple things before I jump off the deep end.

The only real criteria was to loop through some assets and run update() and render() at 60fps in a BS 2D world for a bit without wasting a bunch of time. That's all I want and could ever ask for.

So I took it into my own hands to wast my time instead and looked at C, C++, C#, Zig, Python, Odin, Jai, and even GDScript in something like Godot. You name it.

I never looked at Unity though. Hoping I don't regret that.

While Jai looks epic and C made me feel cool, I felt too stupid to use them effectivly. So after enough time passed I landed on Go of all things.

I originally hated it coming from Java, but as time went on I kept gravitating back to the bizarrely simple and easy to read syntax, even with all its quirks and repetitive not nil error checking.

if err != nil {
    aaaaaahhhhhhhhhhhh!
}

It just does what I need it to do without some weird pattern, straight to an exe. I don't need a f#### Gradle build or a Maven repository or to package a jar thingy in another thingy just because the thingy said so with an annotation that can't load it into memory because your POJO is sad!

(Sorry, repressed feelings. Java is its own thing with pros and cons like any other language.)

I wanted something so simple it was painful. I landed on this guy: https://www.youtube.com/@jarreed0/videos.

In 7 episodes I had all the basics set up. Go with Raylib installed, a basic game loop, and some practice sprites to make your character walk around.

Feels really good so far, but I can already see how much more difficult this would be without a framework like Raylib, or was trying to build something more complex without an out of the box engine like Godot.

Takeaways

If you're in the same position, just trying to figure out what to do or how to build a game, my advice is to not worry about the tech stack or the language.

I think I’ve made a mistake for a while by bouncing around. Take your primary language and dive deep into it to grasp the basics of what you're trying to learn first to see if you even want to do it.

Even if you don't know how to code, just pick a language that seems cool to you because it really does not matter as much as it seems in the beginning.
