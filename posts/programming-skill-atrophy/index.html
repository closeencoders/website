Programming Skill Atrophy

2026-03-17

At my previous job, the pressure to go faster and faster to production without guardrails was mounting quickly.

I knew, as time went on, I would eventually be forced to use AI as much as possible to make important decisions without fully understanding the consequences, while at the same time being held accountable for fixing those same poor decisions in production as fast as possible, as if I were the one who made them.

Shocker, it came true.

I built and maintained card processing applications in Java for a benefits administration company. Not because I wanted to, but because they paid well.

Some of these applications were not written by me. They were written by people who really had no business writing Java applications and would regularly rag on others, saying Java was a terrible language and that everyone around them was lesser because they came from C#.

But then they proceeded to write some of the worst code I have ever seen. When they did not understand what was going wrong, I and a handful of others would have to fix it quietly in the background without throwing a fit, just to avoid scrutiny for trying to push "the Java way". I still don't fully understand what that means to this day.

At the time, I just wanted a paycheck, so I typically yielded to whatever they wanted. My boss would push some bizarre, bastardized standard that was half Java and half C#, and would not let any code go to production unless he felt it looked correct when he was in the mood to review code.

Other times, he would just approve code without looking at it after being pressured by the product team.

I get it, Java has a lot of quirks and people hate on it a lot. I do not really like it either, but it paid the bills and we were using it, like it or not. And there is a general purpose way to write Java applications that another Java developer will not be confused by. There is nothing wrong with adding some off the wall standards here and there if everyone actually follows them.

But guess what, the offshore team quite literally refused to follow them. On almost every commit, this guy was saying the same thing over and over again.

It got to the point of being comical, where I almost relished the dopamine hit of seeing the next pitiful merge request, where the offshore team, mostly comprised of people who only did frontend work but were trying to do Java, tried to make Java look like C#.
